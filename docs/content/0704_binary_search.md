```python
def search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        if nums[mid] > target:
            right = mid - 1
        else:
            left = mid + 1
    return -1


nums = [-1, 0, 3, 5, 9, 12]
target = 9 # 4
print(search(nums, target))
```

这道题目（LeetCode 704. 二分查找）的核心思想是 **“折半规约与范围锁定（Divide and Conquer / Range Reduction）”**。

在算法的兵器谱上，二分查找是**效率**的代名词。如果说暴力搜索是挨个点名的“笨办法”，二分查找就是通过逻辑切分的“降维打击”。

---

### 一、 核心思想：排除法的艺术

二分查找的前提条件只有一个：**数组必须是有序的**。
1.  **腰斩策略**：每次都看数组正中间的那个数。
2.  **非左即右**：
    *   如果中间的数比目标大，那么目标一定在左半边，右半边全部“咔嚓”掉。
    *   如果中间的数比目标小，那么目标一定在右半边，左半边全部“断舍离”。
3.  **指数级收缩**：每一次比较，搜索范围都会直接减半。这种 $O(\log N)$ 的增长速度，让你在 40 亿个数字里找一个数，最多只需要 32 次比较。



---

### 二、 算法逻辑：双指针的包围圈

#### 1. 建立防线
*   `left = 0`, `right = len(nums) - 1`：定义了当前的搜索区间是 **左闭右闭 `[left, right]`**。

#### 2. 收缩阵地（While 循环）
*   只要 `left <= right`，说明区间里还有数字，战斗就没结束。
*   **计算中点**：`mid = (left + right) // 2`。
    *   *注：在某些语言中为了防溢出，会写成 `mid = left + (right - left) // 2`。*

#### 3. 三种结局
*   **刚好撞见**：`nums[mid] == target`，立功回家，返回下标。
*   **猜大了**：`nums[mid] > target`，说明目标在左边，调整右边界 `right = mid - 1`。
*   **猜小了**：`nums[mid] < target`，说明目标在右边，调整左边界 `left = mid + 1`。

#### 4. 颗粒无收
*   如果循环结束还没找到，说明目标不在数组里，返回 `-1`。

---

### 三、 过程模拟 (`nums = [-1, 0, 3, 5, 9, 12], target = 9`)

1.  **第一轮**：`left=0, right=5`。
    *   `mid = (0+5)//2 = 2`。`nums[2]` 是 `3`。
    *   `3 < 9`：猜小了！`left` 移到 `mid + 1 = 3`。
2.  **第二轮**：`left=3, right=5`。
    *   `mid = (3+5)//2 = 4`。`nums[4]` 是 `9`。
    *   **命中！** 返回下标 `4`。

---

### 四、 复杂度分析

*   **时间复杂度：$O(\log N)$**
    *   这是二分查找最迷人的地方。哪怕数据规模翻一倍，它也仅仅是多比对一次而已。
*   **空间复杂度：$O(1)$**
    *   原地查找，只需要定义几个指针变量。

---

### 五、 总结金句

> **“这叫‘极速裁减，逻辑收敛’。二分查找的精髓不在于‘找’，而在于‘舍’。通过利用数组的有序性，我们让每一次比较都产生最大的价值——直接报废一半的可能性。这种对搜索空间的几何级数式压缩，体现了算法设计中对已有规则（有序性）利用到了极致。在处理海量有序数据时，它就是性能的定海神针。”**
