```python
def is_valid(s):
    stack = []
    mapping = {
        ")": "(",
        "]": "[",
        "}": "{",
    }
    for ch in s:
        if ch in mapping:
            if not stack or mapping[ch] != stack[-1]:
                return False
            stack.pop()
        else:
            stack.append(ch)
    return len(stack) == 0


s = "()[]{}"  # true 
s = "([)]"  # false
result = is_valid(s)
print(result)
```

这道题目（LeetCode 20. 有效的括号）的核心思想是 **“后进先出（LIFO, Last In First Out）”**。它是数据结构中 **栈（Stack）** 最经典、最直观的应用场景。

如果说动态规划是在填表格，那么这道题就是在玩一场 **“消消乐”**。

---

### 一、 核心思想：镜像对称与自我救赎



括号匹配的核心在于：**每一个右括号，都必须能和离它最近的左括号“配对成功”并抵消。**

1.  **左括号是“期许”**：当你遇到 `(`、`[`、`{` 时，你就在心里记下一笔，期待后面有一个对应的“另一半”出现。
2.  **右括号是“兑现”**：当你遇到 `)`、`]`、`}` 时，你必须立刻检查最近许下的那个期许是否与它匹配。
3.  **时序性**：因为括号是嵌套的，所以最后许下的期许必须最先被兑现。这完美契合了“栈”的特性。

---

### 二、 算法逻辑：栈的消消乐

#### 1. 建立配对手册（Mapping）
* `mapping` 字典存储了所有右括号到左括号的对应关系。
* **妙处**：我们在字典里只存“右括号”作为 Key。这样在循环时，只要 `ch in mapping`，我们就知道这是一个需要去“兑现”的动作。

#### 2. 遍历字符串
* **如果遇到左括号**：直接把它压入 `stack`（入栈）。这代表一个尚未满足的期许。
* **如果遇到右括号**：
    * **检查落空**：如果此时 `stack` 是空的，说明这个右括号孤苦伶仃，没有左括号能配对，直接 `return False`。
    * **检查错位**：如果栈顶的左括号（`stack[-1]`）不是这个右括号的“亲原配”，说明顺序乱了（比如 `( [ ) ]`），直接 `return False`。
    * **成功抵消**：如果匹配上了，执行 `stack.pop()`，相当于把这个期许从清单中抹除。

#### 3. 最终清算
* 循环结束后，如果 `len(stack) == 0`，说明所有的期许都得到了完美的兑现。如果还剩下什么，说明有人“落单”了。

---

### 三、 过程模拟 (`s = "([)]"`)

1.  **遇到 `(`**：左括号，入栈。`stack = ["("]`
2.  **遇到 `[`**：左括号，入栈。`stack = ["(", "["]`
3.  **遇到 `)`**：是右括号！
    * 查手册：`)` 对应的应该是 `(`。
    * 看栈顶：栈顶现在是 `[`。
    * **冲突！** 期待的是 `[` 的配对，结果来了 `)`。直接返回 `False`。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 我们只需要对字符串进行一次线性扫描。每个字符入栈出栈最多各一次。
* **空间复杂度：$O(N)$**
    * 在最坏的情况下（比如全是左括号 `((((((`），栈的大小会和字符串长度一致。

---

### 五、 总结金句

> **“这叫‘有借有还，分秒不差’。栈在这里充当了一个公正的审计员，它严格遵守着‘最近的期许最先兑现’的原则。任何一个右括号的出现，都是对最近一次左括号的深情回响。如果回响错位，或是无人响应，这段代码构建的逻辑时空就会崩塌。这种对称美，正是栈结构的灵魂所在。”**
