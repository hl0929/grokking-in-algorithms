```python
def queue_reconstruction(people):
    people.sort(key=lambda x: (-x[0], x[1]))
    queue = []
    for p in people:
        queue.insert(p[1], p)
    return queue

people = [[7, 0], [4, 4], [7, 1], [5, 0], [6, 1], [5, 2]]
result = queue_reconstruction(people)
print(result)
```

这道题目（LeetCode 406. 根据身高重建队列）的核心思想是 **“高个子先站位，矮个子后插队”** 的贪心策略。

在解决这类“具有两个维度约束”的问题时，关键在于**通过排序消除一个维度的干扰**，从而让另一个维度变得容易处理。

### 一、 核心思想：屏蔽“矮个子”的影响

如果我们先安排“矮个子”，那么每当后面有一个“高个子”插到他前面时，矮个子要求的 $k$ 值（前面比我高的人数）就会被破坏。

**反过来想：**

* 如果我们**先安排“高个子”**，那么高个子的位置一旦定下来，后面无论我们插入多少个“矮个子”，对高个子来说都是**“隐形”**的。
* 既然矮个子不会影响高个子的 $k$ 值，我们就可以放心地按照身高从高到低的顺序来处理。

---

### 二、 算法逻辑：双重排序 + 插入

代码的逻辑分为极简的两步：

#### 1. 预排序（消除身高维度的不确定性）

* **规则**：按照身高 **降序** 排序；如果身高相同，则按照 $k$ 值 **升序** 排序。
* **目的**：保证当我们处理某个人 $p$ 时，队列中所有已经存在的人，身高都**大于或等于** $p$ 的身高。

#### 2. 贪心插入（处理 $k$ 值的约束）

* **规则**：遍历排序后的数组，将每个人插入到结果队列中索引为 $p[1]$（即他的 $k$ 值）的位置。
* **为什么有效？**
* 因为队列里的人都比当前这个人高。
* 如果他要求前面有 $k$ 个比他高的人，那他就直接坐到索引为 $k$ 的位子上。
* 他坐下后，原本在那里的高个子会往后挪。由于后面挪动的依然是比他高的人，所以他前面的“高个子”数量依然保持为 $k$。



---

### 三、 过程模拟

假设输入：`[[7,0], [4,4], [7,1], [5,0], [6,1], [5,2]]`

1. **排序后**：`[[7,0], [7,1], [6,1], [5,0], [5,2], [4,4]]`
2. **插入 [7,0]**：`[[7,0]]`（插入索引 0）
3. **插入 [7,1]**：`[[7,0], [7,1]]`（插入索引 1）
4. **插入 [6,1]**：`[[7,0], [6,1], [7,1]]`（插入索引 1，[7,1] 被挤到后面去了）
5. **插入 [5,0]**：`[[5,0], [7,0], [6,1], [7,1]]`（插入索引 0）
6. **插入 [5,2]**：`[[5,0], [7,0], [5,2], [6,1], [7,1]]`（插入索引 2）
7. **插入 [4,4]**：`[[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]`（插入索引 4）

---

### 四、 复杂度与总结

* **时间复杂度：$O(N^2)$**
* 排序耗时 $O(N \log N)$。
* 遍历 $N$ 个人，每次执行 `list.insert` 操作在数组中平均需要 $O(N)$ 的移动开销。


* **空间复杂度：$O(N)$** 用于存储排序后的列表和最终的队列。

**总结金句：**

> **“大树先种，小草后栽。小草种在大树前面还是后面，都不会改变大树眼中大树的数量。”** 这种先处理“权重高/约束强”的元素，再利用数据结构的插入特性满足“相对位置”约束的思路，是贪心算法中非常精妙的技巧。
