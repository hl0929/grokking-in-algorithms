```python
def rotate_array(nums, k):
    def reverse(start, end):
        while start < end:
            nums[start], nums[end] = nums[end], nums[start]
            start += 1
            end -= 1
    n = len(nums)
    k = k % n
    reverse(0, n - 1)
    reverse(0, k - 1)
    reverse(k, n - 1)
    
    
nums = [1,2,3,4,5,6,7] 
k = 3 # [5,6,7,1,2,3,4]
rotate_array(nums, k)
print(nums)
```

这道题目（LeetCode 189. 轮转数组）的核心思想是 **“三次反转，移位成真（Triple Inversion）”**。

这是一种非常精妙的数学技巧。虽然我们可以用额外的数组来搬运元素，或者一次次地挪动，但“三次反转法”通过最简单的翻转动作，实现了最复杂的阵位变换。

---

### 一、 核心思想：颠倒中的秩序

想象你有一串项链，你想把后面的一段挪到前面去。
1.  **全局颠倒**：先把整个序列完全反过来。这时候，原本在后面的元素确实到了前面，但它们的**内部顺序**乱了（变成了倒序）。
2.  **局部修正**：
    *   把前 $k$ 个（已经换到前面的部分）再反转一次，恢复它们原本的顺序。
    *   把剩下的部分也再反转一次，恢复它们原本的顺序。



这种“负负得正”的逻辑，让元素在不需要额外空间的情况下，精准地落在了目标位置。

---

### 二、 算法逻辑：三步翻转法

#### 1. 取模优化
*   `k = k % n`：如果数组长度是 7，移动 10 次其实等同于移动 3 次。这是为了防止无谓的整圈循环。

#### 2. 核心工具：`reverse` 函数
*   使用**双指针**从两头向中间靠拢，不断交换元素。这是最基础、最地道的原地翻转写法。

#### 3. 翻转三部曲
*   **第一步：全局大翻转** `reverse(0, n - 1)`
    *   将整个数组彻底倒置。
*   **第二步：前段修复** `reverse(0, k - 1)`
    *   此时前 $k$ 个元素已经是目标元素了，只是顺序是反的，我们要把它们掰回来。
*   **第三步：后段修复** `reverse(k, n - 1)`
    *   剩下的 $n-k$ 个元素同理，再进行一次“负负得正”。

---

### 三、 过程模拟 (`nums = [1,2,3,4,5,6,7], k = 3`)

1.  **初始状态**：`[1, 2, 3, 4, 5, 6, 7]`
2.  **全局反转**：
    *   变成 `[7, 6, 5, 4, 3, 2, 1]`
    *   *你看，5, 6, 7 已经到前面去了，但顺序是 7, 6, 5。*
3.  **反转前 $k$ 个 (前 3 个)**：
    *   把 `[7, 6, 5]` 变成 `[5, 6, 7]`
    *   数组变为：`[5, 6, 7, 4, 3, 2, 1]`
4.  **反转剩余部分 (后 4 个)**：
    *   把 `[4, 3, 2, 1]` 变成 `[1, 2, 3, 4]`
    *   数组变为：`[5, 6, 7, 1, 2, 3, 4]`
5.  **大功告成！**

---

### 四、 复杂度分析

*   **时间复杂度：$O(N)$**
    *   我们虽然反转了三次，但总的元素交换次数依然是线性的。
*   **空间复杂度：$O(1)$**
    *   **这是此法的杀手锏！** 我们没有申请任何额外数组，完全是在原数组上“辗转腾挪”。

---

### 五、 总结金句

> **“这叫‘以退为进，乱中取序’。轮转数组本质上是寻找一种跨度较大的位置交换，而三次反转法则利用了对称性的原理，通过三次‘颠倒黑白’，神奇地让每一个元素都归到了正确的位次。这种不使用任何额外空间的‘原地体操’，不仅是算法之美，更是对内存空间的极致尊重。”**
