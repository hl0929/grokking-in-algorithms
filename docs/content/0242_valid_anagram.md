```python
def valid_anagram(s, t):
    if len(s) != len(t):
        return False
    num_list = [0] * 26
    for ch in s:
        num_list[ord(ch) - ord("a")] += 1
    for ch in t:
        num_list[ord(ch) - ord("a")] -= 1
    for num in num_list:
        if num != 0:
            return False
    return True


s = "anagram" 
t = "nagaram" # true
result = valid_anagram(s, t)
print(result)
```

这道题目（LeetCode 242. 有效的字母异位词）的核心思想是 **“计数抵消”**。如果说“两数之和”是找另一半，那么这道题就像是在检查两份 **“购物清单”** 是否完全一致。

你可以把这个算法想象成一个 **“库存管理员”** 的结账过程。

### 一、 核心思想：天平的平衡

所谓“字母异位词”，本质上是两个字符串**包含的字符种类和每个字符的数量完全相同**，只是顺序不同。

1. **定长哈希（简易表格）**：因为题目通常明确只有 26 个小写字母，我们不需要复杂的哈希表，只需一个长度为 26 的数组（`num_list`）充当计数器。
2. **加减抵消法**：第一个字符串负责“进货”（加分），第二个字符串负责“出货”（减分）。如果最后库存全为 0，说明两边刚好抵消。

---

### 二、 算法逻辑：三步走

以 `s = "rat", t = "car"` 为例：

#### 1. 预处理：门槛检查
首先看长度。如果 `s` 有 3 个字母，`t` 有 4 个，那它们绝对不可能互为异位词。直接拒之门外（`return False`）。

#### 2. 盘点：一增一减
我们准备 26 个抽屉，分别贴上 a 到 z 的标签。
* **扫描 `s`**：每看到一个字母，就在对应的抽屉里放一个球。
    * 看到 'r'，`num_list[17]++`
* **扫描 `t`**：每看到一个字母，就从对应的抽屉里拿走一个球。
    * 看到 'c'，`num_list[2]--`

#### 3. 核账：清点余粮
最后，管理员巡视这 26 个抽屉：
* 如果某个抽屉里还有球（正数），说明 `s` 多了。
* 如果某个抽屉里倒欠球（负数），说明 `t` 多了。
* **只有所有抽屉都空空如也（全为 0），才是完美的异位词。**

---

### 三、 过程模拟 (`s = "anagram", t = "nagaram"`)



1. **处理 `s` ("anagram")**:
    * 'a' 出现 3 次 $\rightarrow$ `list[0] = 3`
    * 'n', 'g', 'r', 'm' 各出现 1 次 $\rightarrow$ 对应位置变为 1。

2. **处理 `t` ("nagaram")**:
    * 遇到 'n' $\rightarrow$ `list[13]` 从 1 变回 0。
    * 遇到 'a'（三次） $\rightarrow$ `list[0]` 从 3 逐步减到 0。
    * ...以此类推。

3. **最终检查**:
    * 遍历 `num_list`，发现全是 0。
    * **判定成功**：返回 `True`。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 我们需要遍历字符串 `s` 和 `t`（长度为 $N$），以及一个固定长度（26）的数组。由于 26 是常数，所以总时间是线性的。
* **空间复杂度：$O(1)$**
    * 虽然我们开辟了空间，但无论字符串多长，数组的大小永远固定为 26。在算法衡量中，这种固定大小的空间被视为常数级别。

---

### 五、 总结金句

> **“暴力法是先排序再比对（$O(N \log N)$），计数法是各拿一本账本。一个负责记账，一个负责平账，账面清零之日，便是匹配成功之时。”**
