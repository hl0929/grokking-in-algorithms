```python
def sort_colors(nums):
    left, right = 0, len(nums) - 1
    curr = 0
    while curr <= right:
        if nums[curr] == 0:
            nums[curr], nums[left] = nums[left], nums[curr]
            curr += 1
            left += 1
        elif nums[curr] == 2:
            nums[curr], nums[right] = nums[right], nums[curr]
            right -= 1
        else:
            curr += 1

nums = [2,0,2,1,1,0] # [0,0,1,1,2,2]
sort_colors(nums)
print(nums)
```

这道题目（LeetCode 75. 颜色分类）的核心思想是 **“双指针（三区域）划分”**，通过 **一次扫描** 将 $O(N \log N)$ 的排序优化为 $O(N)$ 的线性分类。

### 一、 核心思想：各就各位

想象你面前有一排乱七八糟的红色（0）、白色（1）和蓝色（2）小球，你的任务是把它们排好。

三指针法的逻辑是：**“我不需要看清所有球再排序，我只需要在路过每个球时，顺手把它踢到它该去的角落。”**

1.  **左边踢红色**：设置一个 `left` 指针，它左边永远只准待红色（0）。
2.  **右边踢蓝色**：设置一个 `right` 指针，它右边永远只准待蓝色（2）。
3.  **中间留白色**：剩下的白色（1）自然就会被挤在中间。

---

### 二、 角色分配：三个“管理员”

以 `nums = [2, 0, 2, 1, 1, 0]` 为例：

#### 1. `left` 管理员（红色哨兵）
* 它的任务是：**“保卫 0 的领地”**。
* 只要发现 `0`，就跟它换位置，然后把领地边界向右扩大一格。

#### 2. `right` 管理员（蓝色哨兵）
* 它的任务是：**“收容 2 的领地”**。
* 只要发现 `2`，就跟它换位置，然后把领地边界向左收缩一格。

#### 3. `curr` 扫描员（搬运工）
* 它从头走到尾，看每个球的颜色：
    * **看到 0**：扔给 `left`。
    * **看到 2**：扔给 `right`。
    * **看到 1**：管都不管，直接走人（因为它本来就在中间）。

---

### 三、 算法精髓：为什么 `curr` 有时不动？

这是很多初学者最容易晕的地方，也是算法最巧妙的地方：

* **遇到 0 交换后，`curr++`**：因为 `left` 是从左往右扫描过的，换过来的球只可能是 `1`（如果是 `0` 也会被换走），所以 `curr` 可以放心继续走。
* **遇到 2 交换后，`curr` 不动**：因为 `right` 是从右边换过来的，**那个球还没被看过**，万一换过来的是个 `0` 呢？所以 `curr` 必须停在原地，重新审视这个“新球”。

---

### 四、 过程模拟 (`nums = [2, 0, 1]`)

1.  **遇到 2**：
    * `curr` 说：“你是蓝色，去右边！”
    * 与 `right` 交换。数组变成 `[1, 0, 2]`。
    * `right` 左移，但 `curr` 不动（得看看换过来的 `1` 怎么办）。

2.  **遇到 1**：
    * `curr` 说：“白色啊，你就在中间待着吧。”
    * `curr` 直接向右走。

3.  **遇到 0**：
    * `curr` 说：“红色！去左边找 `left` 换位。”
    * 交换后数组变成 `[0, 1, 2]`。

---

### 五、 复杂度分析

* **时间复杂度：$O(N)$**
    * 只需遍历一次数组。每个元素最多被交换两次。
* **空间复杂度：$O(1)$**
    * 只用了三个指针变量，不需要额外的数组空间。

---

### 六、 总结金句

> **“遇到红色（0）往左踢，遇到蓝色（2）往右踢，白色（1）原地不动。这不仅是算法，更是一种‘分而治之’的整理艺术。”**
