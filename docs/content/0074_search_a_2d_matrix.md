```python
def search_matrix(matrix, target):
    m, n = len(matrix), len(matrix[0])
    left, right = 0, m * n - 1
    while left <= right:
        mid = (left + right) // 2
        row = mid // n
        col = mid % n
        if matrix[row][col] == target:
            return True
        elif matrix[row][col] > target:
            right = mid - 1
        else:
            left = mid + 1
    return False


matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]] 
target = 3 #true
result = search_matrix(matrix, target)
print(result)
```

这道题目（LeetCode 74. 搜索二维矩阵）的核心思想是 **“空间降维（Space Flattening）”**。

如果说普通的二分查找是在一根线上找点，那么这道题就是要把一个 **“方阵”** 在大脑中虚构成 **“一根长线”**，从而实现对二维空间的降维打击。

---

### 一、 核心思想：把矩阵拉成直线

这个矩阵有一个非常关键的特性：每行从左到右升序，且每行的第一个数大于前一行的最后一个数。这意味着，如果我们把这个矩阵的所有行首尾相连，它就是一个 **完美的升序数组**。

1.  **逻辑铺平**：我们不需要真的去创建一个新数组（那样会浪费 $O(M \times N)$ 的空间），我们只需要在逻辑上把二维索引映射到一维。
2.  **索引翻译**：
    * 假设矩阵是 $m$ 行 $n$ 列。
    * 一维索引 `mid` 对应的二维坐标是：`行 = mid // n`，`列 = mid % n`。
3.  **精准定位**：有了这个“翻译官”，我们就可以像在普通数组里一样，用 $O(\log(M \times N))$ 的速度横扫整个矩阵。

---

### 二、 算法逻辑：虚拟二分法



#### 1. 设定范围
* 左边界 `left = 0`。
* 右边界 `right = m * n - 1`（总元素个数减 1）。

#### 2. 中点翻译
* 计算 `mid`（一维中点）。
* **核心转换**：
    * `row = mid // n` （整除列数得到它在哪一行）
    * `col = mid % n`  （取模列数得到它在哪一列）
* *这就好比你在电影院找座位：如果你知道每排有 10 个座，那么第 25 号座一定在第 2 排（从 0 开始算）的第 5 个位置。*

#### 3. 标准二分比较
* 如果 `matrix[row][col] == target`，恭喜，找到了。
* 如果大了，往左半段缩；如果小了，往右半段缩。

---

### 三、 过程模拟 (`matrix` = 3x4, `target` = 3)

1.  **初始状态**：`left = 0, right = 11` (总共 12 个数)。
2.  **第一轮**：`mid = (0 + 11) // 2 = 5`。
    * 转换坐标：`row = 5 // 4 = 1`, `col = 5 % 4 = 1`。
    * 检查 `matrix[1][1]`，数值是 **11**。
    * **11 > 3**，说明目标在左边。`right = 5 - 1 = 4`。
3.  **第二轮**：`mid = (0 + 4) // 2 = 2`。
    * 转换坐标：`row = 2 // 4 = 0`, `col = 2 % 4 = 2`。
    * 检查 `matrix[0][2]`，数值是 **5**。
    * **5 > 3**，继续往左缩。`right = 2 - 1 = 1`。
4.  **第三轮**：`mid = (0 + 1) // 2 = 0`。
    * 转换坐标：`row = 0, col = 0`。数值是 **1**。
    * **1 < 3**，往右缩。`left = 0 + 1 = 1`。
5.  **第四轮**：`left = 1, right = 1, mid = 1`。
    * 转换坐标：`row = 0, col = 1`。数值是 **3**。
    * **命中！** 返回 `True`。

---

### 四、 复杂度分析

* **时间复杂度：$O(\log(M \times N))$**
    * 本质上就是对所有元素进行了一次二分查找。
* **空间复杂度：$O(1)$**
    * 我们只用了几个变量来记录边界和坐标，没有产生任何额外的存储负担。

---

### 五、 总结金句

> **“这叫‘心中无方阵，眼中只有线’。通过一套简单的坐标转换公式，我们把层级森严的二维矩阵变成了一马平川的赛道。这种将复杂维度降解为基础问题的思维，正是二分查找最优雅的应用。”**
