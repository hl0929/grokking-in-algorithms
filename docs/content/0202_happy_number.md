```python
def happy_number(n):
    def get_next(x):
        total = 0
        while x > 0:
            remainder = x % 10
            x = x // 10
            total += remainder ** 2
        return total
    
    seen = set()
    while n != 1:
        n = get_next(n)
        if n in seen:
            return False
        seen.add(n)
    return True


n = 19 # true
result = happy_number(n)
print(result)
```

```python
def happy_number(n):
    def get_next(x):
        total = 0
        while x > 0:
            remainder = x % 10
            x = x // 10
            total += remainder ** 2
        return total
    
    slow = n
    fast = get_next(n)
    while fast != 1 and slow != fast:
        slow = get_next(slow)
        fast = get_next(get_next(fast))
    return fast == 1


n = 19 # true
result = happy_number(n)
print(result)
```

“快乐数”（LeetCode 202）的核心思想是 **“环路检测”**。它将一个看似无穷无尽的数字变换过程，转化为一个 **“是否会陷入死循环”** 的拓扑问题。

你可以把这个算法想象成一场 **“数字的轮回转世”**。

### 一、 核心思想：归一还是轮回？

在快乐数的定义中，每一个数字都在不断进行“自我的平方和”变换。这个过程只有两种结局：
1. **修成正果**：数字最终变成了 **1**，从此永远停留在 1（因为 $1^2 = 1$），这种数字是“快乐”的。
2. **堕入轮回**：数字进入了一个**不包含 1 的死循环**，它在几个特定的数字之间永不停歇地打转，这种数字是“不快乐”的。

* **逻辑基石**：由于任意位数的数字经过平方和变换后，数值会迅速缩小到一个很小的范围内（例如 3 位数最大 999 变换后也才 243），所以它**不可能无限增大**，必然会形成一个闭环。

---

### 二、 算法逻辑：建立“记忆档案”

以 `n = 19` 为例：

#### 1. 记录足迹（HashSet）
我们准备一个“备忘录”（哈希表），每算出一个新数字，就去查一下：**“这个数字我以前见过吗？”**

#### 2. 逻辑分支
* **看到 1**：大功告成，返回 `True`。
* **看到旧数字**：不好！我们回到了曾经走过的地方。这意味着我们已经掉进了一个死循环，永远等不到 1 了，返回 `False`。
* **看到新数字**：把它记在备忘录里，继续转世。

---

### 三、 过程模拟 (`n = 19` vs `n = 2`)

#### 案例 A：快乐的 19
1.  **19**：$1^2 + 9^2 = 82$。记录 `[19]`。
2.  **82**：$8^2 + 2^2 = 68$。记录 `[19, 82]`。
3.  **68**：$6^2 + 8^2 = 100$。记录 `[19, 82, 68]`。
4.  **100**：$1^2 + 0^2 + 0^2 = 1$。**到达终点！返回 True**。

#### 案例 B：忧郁的 2
1.  **2** $\to$ 4 $\to$ 16 $\to$ 37 $\to$ 58 $\to$ 89 $\to$ 145 $\to$ 42 $\to$ 20 $\to$ **4**
2.  **惊变**：当算出第二个 **4** 时，查备忘录发现它出现过！
3.  **结论**：数字 2 已经掉进了 `[4, 16, 37, 58, 89, 145, 42, 20]` 这个无尽的轮回中。**返回 False**。

---

### 四、 复杂度分析

* **时间复杂度：$O(\log N)$**
    * 虽然看起来步骤很多，但数字缩小的速度呈指数级。对于非常大的数，它的平方和会迅速降到 1000 以内，之后的变换步数是有限的。
* **空间复杂度：$O(\log N)$**
    * 我们需要哈希表来存储路径上的数字。由于数字范围被迅速限制，存储的元素个数其实非常少。

---

### 五、 总结金句

> **“快乐数算法是一场关于因果的审判。每一个数字都在努力通过不断的自我解构（平方和）来实现升华。如果它能斩断过去的牵绊（不重复）并归于初心（1），它便是快乐的；如果它始终逃不出过去的阴影（重复），那它将永陷轮回。”**
