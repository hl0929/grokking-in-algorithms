
[1. Two Sum](https://leetcode.com/problems/two-sum)

[0001_two_sum](./html/0001_two_sum.html ":include :type=iframe")

```python
def two_sum(nums, target):
    hashmap = dict()
    for i, num in enumerate(nums):
        if num not in hashmap:
            hashmap[target - num] = i
        else:
            return [hashmap[num], i]


nums = [3, 2, 4]
target = 6
result = two_sum(nums, target)
print(result)
```

这道题目（LeetCode 1. 两数之和）的核心思想是 **“空间换时间”**，通过 **哈希表（Hash Map）** 将 $O(N^2)$ 的暴力搜索优化为 $O(N)$ 的线性扫描。

你可以把这个算法想象成一个 **“相亲配对”** 的过程。

### 一、 核心思想：寻找“另一半”

传统的暴力解法是：每看到一个数，就去后面所有数里挨个问：“你是我的另一半（凑成 target）吗？”
而哈希表法的逻辑是：**“如果我现在没找到另一半，我就在登记表上留下我需要的数字，等后面的人来认领。”**

1. **逆向思维**：我们不去找 `nums[j]`，而是计算出当前数字 `nums[i]` 理想的补数 `complement = target - nums[i]`。
2. **记忆化搜索**：哈希表就像一个“备忘录”，记录了我们**期望遇到的数字**以及**谁在等它**。

---

### 二、 算法逻辑：三步走

以 `nums = [3, 2, 4], target = 6` 为例：

#### 1. 询问：有人在等我吗？

当遍历到当前数字 `num` 时，先去哈希表里查：**“以前有没有人登记过，说他正缺 `num` 这个数？”**

* 如果有，说明配对成功，直接返回那个人的索引和当前索引。

#### 2. 登记：我需要谁？

如果没有人在等 `num`，说明此时还没找到配对。

* 我们计算出自己的需求：`need = target - num`。
* 在哈希表中存入：`hashmap[need] = i`。
* *含义：如果以后有人看到了数字 `need`，请记住索引 `i` 正在等它。*

#### 3. 结果产出

一旦哈希表中出现了当前数字，算法立即停止并返回结果，保证了它是最先被找到的一组解。

---

### 三、 过程模拟 (`nums = [3, 2, 4], target = 6`)

1. **遇到 3** (idx 0)：
* 表中没 3。
* 计算补数：`6 - 3 = 3`。
* 登记：`{3: 0}` （如果后面看到 3，找索引 0）。


2. **遇到 2** (idx 1)：
* 表中没 2。
* 计算补数：`6 - 2 = 4`。
* 登记：`{3: 0, 4: 1}` （如果后面看到 4，找索引 1）。


3. **遇到 4** (idx 2)：
* 去表里查：**发现 4 在表里！** 对应索引是 1。
* **匹配成功**：返回 `[1, 2]`。



---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
* 我们只对数组进行了一次遍历。
* 哈希表的查找（`in` 操作）和插入操作在平均情况下都是 $O(1)$。


* **空间复杂度：$O(N)$**
* 最坏情况下（比如最后一对才匹配成功），我们需要在哈希表中存储 $N-1$ 个元素。
