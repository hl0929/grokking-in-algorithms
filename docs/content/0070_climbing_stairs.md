```python
def climb_stairs(n):
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b


n = 5
result = climb_stairs(n)
print(result)
```

这道题目（LeetCode 70. 爬楼梯）的核心思想是 **“状态转移（State Transition）”**。在算法世界里，它通常被称为 **“动态规划（Dynamic Programming）”** 的入门第一课。

虽然它听起来像是一个数学谜题，但本质上它是一个关于 **“过去决定未来”** 的故事。

---

### 一、 核心思想：寻找最后一步

想象你站在第 $n$ 级台阶上，回想一下你是怎么上来的：
1.  **从 $n-1$ 级跳上来**：你迈了 **1** 个台阶。
2.  **从 $n-2$ 级跳上来**：你迈了 **2** 个台阶。



**结论：** 到达第 $n$ 级的总方法数，就是到达“倒数第一步”和“倒数第二步”的方法数之和。这就是著名的 **斐波那契数列（Fibonacci Sequence）** 在生活中的变种。

---

### 二、 算法逻辑：空间优化的滚动迭代

#### 1. 基础认知（Base Case）
* 如果楼梯只有 1 级，你只有 1 种走法。
* 如果楼梯有 2 级，你有 2 种走法（1+1 或直接跨 2 步）。

#### 2. 状态转移
* 每一级台阶的方法数 $f(n) = f(n-1) + f(n-2)$。

#### 3. 滚动更新（这里的代码精髓）
* 我们不需要创建一个巨大的数组来存每一层的结果。
* 我们只用了两个变量 `a`（前前一步）和 `b`（前一步）。
* 每次循环，就像是两个人在玩 **“接力跑”**：新的 `b` 变成了前两者的和，而旧的 `b` 则退位变成了 `a`。

---

### 三、 过程模拟 (`n = 5`)

1.  **初始状态**：`a = 1` (1级), `b = 2` (2级)。
2.  **计算第 3 级**：
    * `new_b = 1 + 2 = 3`
    * 状态变为：`a = 2, b = 3`
3.  **计算第 4 级**：
    * `new_b = 2 + 3 = 5`
    * 状态变为：`a = 3, b = 5`
4.  **计算第 5 级**：
    * `new_b = 3 + 5 = 8`
    * 状态变为：`a = 5, b = 8`
5.  **返回结果**：8。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 我们只需要从 3 扫到 $N$，每个数字只算一次，速度极快。
* **空间复杂度：$O(1)$**
    * **这是本代码的亮点！** 相比于递归或者使用 O(N) 空间的数组，我们只用了两个变量，极其节省内存。

---

### 五、 总结金句

> **“这叫‘步步为营，记忆迭代’。爬楼梯看似复杂，但只要你明白‘未来的可能建立在过去的积累之上’，你就不需要去数每一条路径。通过两个变量的左右横跳，我们把指数级的可能性降维成了线性的加法。这正是动态规划‘化繁为简’的魅力所在。”**

### 💡 思考题
既然爬楼梯可以用动态规划解决，那如果规则变了：**“你不能连续两次迈 2 个台阶”**，你觉得这个状态转移方程该怎么修改呢？这可是大厂面试中常见的动态规划变种题哦！
