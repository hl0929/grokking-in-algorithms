class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
        
        
def detect_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:
        return None
    node = head
    while node != slow:
        node = node.next
        slow = slow.next
    return node

这道题目（LeetCode 142. 环形链表 II）是上一题的**进阶版**。如果说上一题只是为了证明“圈子”的存在，那么这一题的任务就是：**精准定位这个圈子的“入口”在哪里。**

我们要用的依然是 **“快慢指针”**，但这次需要一点点数学上的“时空交汇”逻辑。

---

### 一、 核心思想：重逢在起点

想象你在一个带有长长引道的环形操场跑步：
1.  **第一阶段（追及）**：兔子和乌龟在环里相遇了。这证明了有环。
2.  **第二阶段（揭秘）**：相遇的那一刻，我们把乌龟送回整个路径的**起点**，而兔子原地待命（速度降到和乌龟一样）。
3.  **最终奇迹**：当两人再次同步出发，他们**下一次相遇的地方，一定就是环的入口。**

---

### 二、 算法逻辑：数学的浪漫对冲



#### 1. 第一步：寻找“撞击点”
* `slow` 走 1 步，`fast` 走 2 步。
* 如果 `fast` 走到头了，说明没环，直接返回 `None`。
* 如果 `slow == fast`，说明他们相遇了，**记住这个位置**。

#### 2. 第二步：同步归位
* 把一个新的指针 `node` 放回 `head`（起点）。
* 之前的 `slow` 留在刚才的相遇点。
* **同步走**：`node` 走一步，`slow` 也走一步。

#### 3. 第三步：入场相遇
* 当 `node == slow` 时，它们指向的那个节点就是环的起始位置。

---

### 三、 过程模拟（推荐 Testcase: `3 -> 2 -> 0 -> -4 -> (回到2)`）

这个案例比 `1 -> 2` 更具代表性，因为它有一个明显的“引道”。

1.  **第一阶段**：
    * `slow` 每次走 1，`fast` 每次走 2。
    * 经过几次追逐，它们最终会在 **-4** 这个节点相遇。
2.  **第二阶段**：
    * 乌龟 `node` 回到起点 **3**。
    * 兔子 `slow` 留在 **-4**。
    * **同步出发**：
        * `node` 从 3 走到 **2**。
        * `slow` 从 -4 走到 **2**（根据环的指向，-4 的下一个是 2）。
3.  **判定**：
    * 两人在 **2** 再次碰面！而 **2** 恰恰就是环的入口。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 虽然分成了两个阶段，但本质上每个指针走过的总路程都不会超过 $2N$。
* **空间复杂度：$O(1)$**
    * 这种方法最优雅的地方在于：**不需要开辟额外的哈希表**来存储走过的节点。

---

### 五、 总结金句

> **“这就像是一个数学上的宿命约定：当你和你在环中偶遇的人同时以相同的步伐出发——一个从最初的起点开始，一个从相遇的终点开始——你们终将在那个让你迷失方向的入口处重逢。这就是算法里最美妙的‘步调一致’。”**

---

### 为什么这个数学逻辑成立？
简单来说，假设起点到环入口距离为 $a$，环入口到相遇点距离为 $b$，相遇点再回到入口距离为 $c$。
因为兔子速度是乌龟两倍，通过推导 $2(a+b) = a+b+n(b+c)$，最终可以得出 **$a = c$**（忽略掉多跑的圈数）。所以，从起点出发的乌龟走完 $a$，和从相遇点出发的乌龟走完 $c$，正好会在入口撞个满怀！
