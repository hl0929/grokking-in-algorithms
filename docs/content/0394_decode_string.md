```python
def decode_string(s):
    stack = []
    cur_str = ""
    cur_num = 0
    for ch in s:
        if ch.isdigit():
            cur_num = cur_num * 10 + int(ch)
        elif ch == "[":
            stack.append([cur_str, cur_num])
            cur_str = ""
            cur_num = 0
        elif ch == "]":
            prev_str, num = stack.pop()
            cur_str = prev_str + cur_str * num
        else:
            cur_str += ch
    return cur_str


s = "3[a2[c]]"  # accaccacc
result = decode_string(s)
print(result)
```

这道题目（LeetCode 394. 字符串解码）的核心思想是 **“层级嵌套与现场还原（Nested Reconstruction）”**。

它在处理括号嵌套时的逻辑，与“有效括号”极其相似。不同的是，这里我们不仅要匹配括号，还要在括号闭合的一瞬间，完成一场 **“局部字符串的指数级扩容”**。

---

### 一、 核心思想：记忆的暂时“封存”



当你遇到 `3[a2[c]]` 这种结构时，你的大脑会自动进入一种“嵌套模式”：
1.  **外部等待**：当你看到 `3[` 时，你知道后面的内容要重复 3 次，但具体是什么还没确定。
2.  **现场封存**：为了处理内部的 `2[c]`，你必须把当前的“倍数 3”和“已经写好的前缀”先存进大脑的一个角落（栈）。
3.  **局部处理**：先搞定最内层的 `cc`。
4.  **读取还原**：当最内层的 `]` 出现时，从大脑角落里取出刚才封存的倍数和前缀，把 `cc` 合并进去。

---

### 二、 算法逻辑：状态的压入与弹出

#### 1. 变量分工
* `cur_str`：当前正在构建的字符串片段（像是你正在写字的纸）。
* `cur_num`：当前累积的重复倍数（处理像 `100[` 这种多位数）。
* `stack`：用来存放 **“还没处理完的外部环境”**。

#### 2. 四种字符的博弈
* **数字 (`isdigit`)**：由于数字可能是多位，我们要不断 `* 10 + int(ch)` 拼接。
* **左括号 (`[`)**：这是“嵌套”的开始。
    * **动作**：把当前写的 `cur_str` 和攒的 `cur_num` 一起扔进栈里。
    * **重置**：清空纸张（`cur_str = ""`）和数字，开始记录括号里面的内容。
* **右括号 (`]`)**：这是“解码”的时刻。
    * **动作**：从栈里弹出 `prev_str`（之前的老前缀）和 `num`（重复次数）。
    * **公式**：`新字符串 = 老前缀 + 现在的片段 * num`。
* **字母**：普通字符，直接写在 `cur_str` 上。

---

### 三、 过程模拟 (`s = "3[a2[c]]"`)

1.  **遇到 `3[`**：
    * 栈存入 `["", 3]`，`cur_str` 清空。
2.  **遇到 `a`**：
    * `cur_str = "a"`。
3.  **遇到 `2[`**：
    * 栈再存入 `["a", 2]`，`cur_str` 再次清空。
4.  **遇到 `c`**：
    * `cur_str = "c"`。
5.  **遇到第一个 `]`**：
    * 弹出 `["a", 2]`。
    * 计算：`"a" + "c" * 2 = "acc"`。
    * 现在 `cur_str` 变成了 `"acc"`。
6.  **遇到第二个 `]`**：
    * 弹出 `["", 3]`。
    * 计算：`"" + "acc" * 3 = "accaccacc"`。
7.  **最终结果**：`"accaccacc"`。

---

### 四、 复杂度分析

* **时间复杂度：$O(S)$**
    * $S$ 是解码后字符串的总长度。虽然我们只遍历了一次原字符串，但构建最终字符串的过程需要处理所有字符。
* **空间复杂度：$O(N)$**
    * $N$ 是嵌套的深度。栈的大小取决于你有多少层括号嵌套。

---

### 五、 总结金句

> **“这叫‘有备而来，层层剥离’。每一个左括号都是一次对现状的‘存档’，而每一个右括号都是一次‘读档并扩建’。通过栈的保护，我们成功地将复杂的嵌套逻辑转化为了局部的字符串拼接。这种不断回到过去、建设未来的逻辑，正是处理递归型结构的核心密码。”**
