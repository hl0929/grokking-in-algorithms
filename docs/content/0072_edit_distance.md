```python
def edit_distance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1):
        dp[i][0] = i  # 删除
    for j in range(n + 1):
        dp[0][j] = j  # 插入
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,     # 删除
                    dp[i][j - 1] + 1,     # 插入
                    dp[i - 1][j - 1] + 1, # 替换
                )
    return dp[m][n]


word1 = "horse" 
word2 = "ros"  # 3
result = edit_distance(word1, word2)
print(result)
```

这道题目（LeetCode 72. 编辑距离）的核心思想是 **“成本最小化（Cost Minimization）”**。在算法领域，它被用于衡量两个字符串的 **莱文斯坦距离（Levenshtein distance）**，是拼写纠错和基因序列分析的灵魂算法。

如果说“最长公共子序列（LCS）”是在找共同点，那么“编辑距离”就是在计算 **“整容成本”**。

---

### 一、 核心思想：文字的“整容计划”

想象你要把字符串 $A$ 变成字符串 $B$，你手里有三种手术刀：
1.  **插入（Insert）**：凭空变出一个字符。
2.  **删除（Delete）**：切掉一个字符。
3.  **替换（Replace）**：把一个字符整形成另一个字符。

每一刀的成本都是 **1**。我们的目标是：用最少的刀数，完成从 $A$ 到 $B$ 的蜕变。

---

### 二、 算法逻辑：构建二维成本地图



#### 1. 建立空地（初始化）
* `dp[m+1][n+1]`：`dp[i][j]` 代表把 `word1` 的前 `i` 个字符变成 `word2` 的前 `j` 个字符最少需要多少步。
* **第一行（空变有）**：把空字符串变成长度为 `j` 的字符串，只能靠**插入**，步数就是 `j`。
* **第一列（有变空）**：把长度为 `i` 的字符串变成空，只能靠**删除**，步数就是 `i`。

#### 2. 状态转移：精准的手术方案
我们来到棋盘的某个点 `(i, j)`，看这两个字符 `word1[i-1]` 和 `word2[j-1]`：

* **天生一对（字符相同）**：
    * `dp[i][j] = dp[i-1][j-1]`。
    * **逻辑**：不需要动手术，成本直接继承自去掉这两个字符后的状态。
* **需要整形（字符不同）**：我们要从三个方向看哪种手术最省钱：
    1.  **左方（插入）**：`dp[i][j-1] + 1`
    2.  **上方（删除）**：`dp[i-1][j] + 1`
    3.  **斜上方（替换）**：`dp[i-1][j-1] + 1`
    * **逻辑**：`dp[i][j] = min(插入, 删除, 替换)`。

---

### 三、 过程模拟 (`word1="horse", word2="ros"`)

1.  **处理 'h' 变 'r'**：不匹配。我们可以选择替换（把 h 换成 r），成本从 `dp[0][0]` 转移来，变为 1。
2.  **处理 'ho' 变 'ro'**：末尾都是 'o'，完美匹配！成本直接继承自 'h' 变 'r' 的结果，依然是 1。
3.  **最后...**：随着填表推进，当我们要把 `horse` 彻底变成 `ros` 时，最优解可能是：
    * `horse` -> `rorse` (替换 h)
    * `rorse` -> `rose` (删除 r)
    * `rose` -> `ros` (删除 e)
    * **总成本**：3。

---

### 四、 复杂度分析

* **时间复杂度：$O(M \times N)$**
    * $M$ 和 $N$ 是两个单词的长度。我们需要遍历整张 $M \times N$ 的表格，每个格子的逻辑都是常数级的。
* **空间复杂度：$O(M \times N)$**
    * 我们开辟了一个二维数组。
    * *小贴士：通过滚动数组，空间可以优化到 $O(N)$。*

### 五、 总结金句

> **“这叫‘步步为营，最小代价’。编辑距离不再是漫无目的的尝试，它通过一张动态规划表，穷举了所有可能的整形方案，并永远选择最省钱的那一条路。它告诉我们，从一个词变成另一个词，虽然路途遥远，但只要每一步都做出当前最理性的选择，终能抵达终点。”**

### 💡 深度思考
你会发现，**替换** 操作对应的方向是“斜上方”，而 **插入** 和 **删除** 分别对应“左方”和“上方”。这种几何关系在所有的双序列动态规划题（如 LCS, 距离计算）中几乎是通用的。你能试着总结一下，为什么“删除”对应的是 `dp[i-1][j]` 吗？（提示：想想 `i-1` 代表哪个词变短了？）
