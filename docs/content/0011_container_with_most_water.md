```python
def max_area(height):
    area = 0
    left, right = 0, len(height) - 1
    while left < right:
        w = right - left
        if height[left] > height[right]:
            h = height[right]
            right -= 1
        else:
            h = height[left]
            left += 1
        area = max(area, w * h)
    return area


height = [1,8,6,2,5,4,8,3,7] # 49 
result = max_area(height)
print(result)
```

“盛最多水的容器”（LeetCode 11）的核心思想是 **“贪心策略”** 与 **“双指针收缩”**。它将 $O(N^2)$ 的暴力穷举优化为了 $O(N)$ 的线性扫描。

你可以把这个算法想象成一个 **“寻找最优木桶”** 的淘汰赛。

### 一、 核心思想：永远放弃“短板”

在 $O(N^2)$ 的暴力解法中，我们会固定左边，挨个去试右边。但这种方法做了很多“无用功”。

双指针法的逻辑是：**“既然水箱的容量由短板决定，那么除非我们能换掉当前的短板，否则无论怎么移动长板，容量只会因为宽度减小而缩水。”**

1. **木桶效应**：容器能装多少水，取决于**矮的那根柱子**。
2. **排除法**：如果我们移动了长的那根柱子，宽度 $W$ 减小了，而高度 $H$ 绝不可能超过当前的短板。结论：**移动长板，面积必减。**
3. **唯一的希望**：只有移动短板，才有可能在宽度缩小的代价下，换来一个更高的高度，从而逆袭增加面积。

---

### 二、 算法逻辑：两头包抄

以 `height = [1, 8, 6, 2, 5, 4, 8, 3, 7]` 为例：

#### 1. 初始状态：横跨两端
我们把左右指针 `L` 和 `R` 分别放在数组的最开头和最末尾。此时**宽度（底边）达到最大**。

#### 2. 计算并更新
计算当前 `(R - L) * min(height[L], height[R])`。如果比记录的最大面积大，就更新它。

#### 3. 淘汰短板
对比 `height[L]` 和 `height[R]`：
* 如果 `L` 比较矮，执行 `L++`（放弃当前的短板，向右找更高的）。
* 如果 `R` 比较矮，执行 `R--`（放弃当前的短板，向左找更高的）。

#### 4. 循环终止
当 `L` 和 `R` 相遇时，比赛结束，我们已经排除了所有不可能产生更大面积的组合。

---

### 三、 过程模拟 (`height = [1, 8, 6, 2, 5, 4, 8, 3, 7]`)

1. **初始 L=0(高1), R=8(高7)**:
   * 宽 = 8，高 = min(1, 7) = 1。
   * 面积 = 8。
   * **决策**：1 比 7 矮，**L 右移**。

2. **L=1(高8), R=8(高7)**:
   * 宽 = 7，高 = min(8, 7) = 7。
   * 面积 = 49（目前最大！）。
   * **决策**：7 比 8 矮，**R 左移**。

3. **L=1(高8), R=7(高3)**:
   * 宽 = 6，高 = min(8, 3) = 3。
   * 面积 = 18。
   * **决策**：3 比 8 矮，**R 左移**。

...(中间步骤略)...

4. **最终结果**：在这个过程中，我们锁定了 `L=1` 和 `R=6`（高度分别为 8 和 8，宽 5，面积 40）等组合，但最终的 Global Max 停留在 49。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
  * 左右指针像两支军队相向而行，每个元素最多被访问一次。
* **空间复杂度：$O(1)$**
  * 我们只用了几个变量记录指针位置和最大面积，不需要额外的存储空间。

---

### 五、 总结金句

> **“人生和水箱一样，高度往往取决于你的短板。当宽度在无可奈何地流逝（Width 减小）时，你唯一的出路就是不断挑战并替换掉当前的短板，去寻找那个更高的可能。”**
