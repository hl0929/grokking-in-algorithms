
[448. Find All Numbers Disappeared in an Array](https://leetcode.cn/problems/find-all-numbers-disappeared-in-an-array)

```python
def find_disappeared_number(nums):
    for num in nums:
        idx = abs(num) - 1
        if nums[idx] > 0:
            nums[idx] = -nums[idx]
    res = []
    for i, num in enumerate(nums):
        if num > 0:
            res.append(i + 1)
    return res


nums = [4,3,2,7,8,2,3,1] # [5,6] 
result = find_disappeared_number(nums)
print(result)
```

这道题目（LeetCode 448. 找到所有数组中消失的数字）的思路非常惊艳。如果说前面的题目是“开辟新仓库”，那么这道题的核心思想就是 **“原地作标记”**，实现 **$O(1)$ 的额外空间复杂度**。

你可以把这个算法想象成一场 **“回老家报到”** 的过程。

---

### 一、 核心思想：身体虽然不在，但座位要打叉

假设有 $n$ 个房间（数组下标 $0$ 到 $n-1$），本来应该住着编号为 $1$ 到 $n$ 的 $n$ 个人。现在有些号码的人重了，导致有些号码的人没来。

1.  **利用自身空间**：我们不买新的记事本，直接在原来的数组上做记号。
2.  **正负号标记法**：如果编号为 `x` 的人回来了，我们就去第 `x-1` 个房间，把那里的房客衣服颜色**反转（变成负数）**。负号就代表：“这个房间对应的编号已经报到过了”。

---

### 二、 算法逻辑：两步走

以 `nums = [4, 3, 2, 7, 8, 2, 3, 1]` 为例：

#### 1. 第一轮：点名并“打叉”
我们遍历数组，每看到一个数字，就把它当成一个“地址”。
* 看到数字 `4`：说明编号 `4` 出现了。我们去 `index = 3` 的位置，把那里的数变成负数。
* 注意：因为数字可能已经被别人改成了负数，所以我们要用 `abs(num)` 来获取它的原始值。

#### 2. 第二轮：检查谁没报到
再次遍历数组，看看哪些位置的数字**依然是正数**。
* 如果第 `i` 个位置是正数，说明**从来没有人**以 `i+1` 这个编号回来报到过。
* 那么，`i+1` 就是那个消失的数字。



---

### 三、 过程模拟 (`nums = [4, 3, 2, 7, 8, 2, 3, 1]`)

1.  **遇到 4**: 去索引 3，把 `7` 变 `-7`。
2.  **遇到 3**: 去索引 2，把 `2` 变 `-2`。
3.  **遇到 2**: 去索引 1，把 `3` 变 `-3`。
4.  **遇到 7**: 去索引 6，把 `3` 变 `-3`。
5.  **遇到 8**: 去索引 7，把 `1` 变 `-1`。
6.  **遇到 2 (重复)**: 去索引 1，发现已经是 `-3` 了，保持不变。
7.  **遇到 3 (重复)**: 去索引 2，发现已经是 `-2` 了，保持不变。
8.  **遇到 1**: 去索引 0，把 `4` 变 `-4`。

**最终数组状态：**
`[-4, -3, -2, -7, 8, 2, -3, -1]`
索引：`0, 1, 2, 3, 4, 5, 6, 7`

此时，只有索引 **4** 和 **5** 位置上的数字是正数。对应的编号就是 `4+1=5` 和 `5+1=6`。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 我们虽然遍历了两次数组，但它们是并列的，总时间依然是线性的。
* **空间复杂度：$O(1)$**
    * **这是本题最亮眼的地方！** 除了返回结果的列表外，我们没有使用任何额外的哈希表或数组，直接“白嫖”了原数组的空间。

---

### 五、 总结金句

> **“这叫‘走过必留下痕迹’。我们把数组当成一排宿舍，回来的每个数字都去自己对应的宿舍门上贴个‘已到’的负号。最后谁家门上没贴纸，谁就是那个失踪人口。”**
