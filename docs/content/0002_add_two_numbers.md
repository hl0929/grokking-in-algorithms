
[2. Add Two Numbers](https://leetcode.cn/problems/add-two-numbers/)

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
        
        
def add_two_number(l1, l2):
    dummy = ListNode()
    curr = dummy
    carry = 0
    while l1 or l2 or carry:
        v1 = l1.val if l1 else 0
        v2 = l2.val if l2 else 0
        total = v1 + v2 + carry
        carry = total // 10
        curr.next = ListNode(total % 10)
        curr = curr.next
        l1 = l1.next if l1 else None
        l2 = l2.next if l2 else None
    return dummy.next


# l1 = [2,4,3], l2 = [5,6,4] #[7,0,8]
l1 = ListNode(2)
a = ListNode(4)
b = ListNode(3)
l2 = ListNode(5)
c = ListNode(6)
d = ListNode(4)
l1.next = a
a.next = b
l2.next = c
c.next = d
result = add_two_number(l1, l2)
while result:
    print(result.val)
    result = result.next
```


这道题目（LeetCode 2. 两数相加）的核心思想是 **“模拟竖式加法”**。

这题最精妙的地方在于：它利用了链表 **“逆序存储”** 的特点（个位在头，高位在后），这恰好符合我们平时做加法从低位到高位的习惯。

---

### 一、 核心思想：链表上的“进位器”

你可以把这个算法想象成一个 **“流水线加工过程”**：

1.  **同步对齐**：两个链表 $l1$ 和 $l2$ 就像两条平行的传送带，我们同时读取当前的“零件”（数字）。
2.  **满十进一**：这跟我们在纸上做加法一模一样。如果两个数加起来超过了 10，我们就得把个位数留下，把那个“1”带到下一组加工。
3.  **哑节点占位**：由于我们不知道结果的第一位是多少，先用一个 `dummy` 节点作为起点的“钩子”，方便后续延伸出整条结果链表。

---

### 二、 算法逻辑：三位一体的加法



#### 1. 变量初始化
* `dummy`：结果链表的假头，最后返回 `dummy.next`。
* `curr`：就像一个移动的针头，负责把新算出来的数字节点一个一个串起来。
* `carry`：专门用来存那个“进位”。

#### 2. 循环条件：一个都不能少
* `while l1 or l2 or carry:` 
* 这个条件非常严谨。只要 $l1$ 没走完，或者 $l2$ 没走完，或者最后还有一个 **进位（carry）** 没处理，流水线就不能停。

#### 3. 计算逻辑：提取、求和、更新
* **提取**：如果某个链表已经空了（走到头了），我们就把它的值当做 **0** 处理。
* **求和**：当前总和 = $v1$ + $v2$ + 进位。
* **分拆**：总和除以 10 得到新的进位，总和取模 10 得到当前位的值。
* **移动**：各就各位，向后移一个节点。

---

### 三、 过程模拟 (`2->4->3` + `5->6->4`)

1.  **第一轮（个位）**：
    * $2 + 5 + 0$（进位）$= 7$。
    * 进位变为 0，创建节点 **7**。结果：`7`。
2.  **第二轮（十位）**：
    * $4 + 6 + 0$（进位）$= 10$。
    * 进位变为 1，创建节点 **0** ($10\%10$)。结果：`7 -> 0`。
3.  **第三轮（百位）**：
    * $3 + 4 + 1$（进位）$= 8$。
    * 进位变为 0，创建节点 **8**。结果：`7 -> 0 -> 8`。
4.  **结束**：$l1, l2$ 均空且 `carry` 为 0，停机。

---

### 四、 复杂度分析

* **时间复杂度：$O(\max(M, N))$**
    * $M$ 和 $N$ 分别是两个链表的长度。我们只需要把最长的那个链表完整地走一遍。
* **空间复杂度：$O(1)$**
    * 撇开存储结果的链表不谈，我们只用了 `carry` 和几个指针变量。这是极其节省空间的做法。

---

### 五、 总结金句

> **“这道题是‘化繁为简’的典型。虽然链表看起来比数字复杂，但由于它是逆序的，反而帮我们省去了‘对齐末位’的烦恼。只要守住那个‘进位（carry）’，再长的数字链表在流水线下都会被拆解成最基础的 0-9 加法。”**

这种“模拟法”是算法面试中的基本功。掌握了这道题，你可以试着去挑战一下它的进阶版——**“如果链表是正序存储的（高位在前）”**（LeetCode 445），那可能就需要用到 **栈（Stack）** 或者 **链表翻转** 的技巧了！
