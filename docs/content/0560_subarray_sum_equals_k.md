
[560. Subarray Sum Equals K](https://leetcode.cn/problems/subarray-sum-equals-k)

[0560_subarray_sum_equals_k](./html/0560_subarray_sum_equals_k.html ":include :type=iframe")

```python
def subarray_sum(nums, k):
    prefix = 0
    cnter = 0
    sum_dict = {0: 1}
    for num in nums:
        prefix += num
        if prefix - k in sum_dict:
            cnter += sum_dict[prefix - k]
        sum_dict[prefix] = sum_dict.get(prefix, 0) + 1
    return cnter


nums = [3, 4, 7, -2, 2, 1, 4, 2]
k = 7 # 6
result = subarray_sum(nums, k)
print(result)
```

这道题目（LeetCode 560. 和为 K 的子数组）是前缀和类题目中最经典的一道，它的核心思想是 **“将区间问题转化为点差问题”**。


### 一、 核心思想：寻找“影子”

想象你正在一条数轴上行走，步数累加的结果就是 **前缀和（Prefix Sum）**。如果你想知道一段路程的和是否等于 $K$，你不需要回头去量，只需要看你现在的坐标 $P$ 和以前经历过的某个坐标 $P_{old}$ 之间的差值。

**逻辑精髓：** “如果现在的总和是 $10$，目标 $K$ 是 $7$，那么只要我以前到过坐标 $3$（即 $10 - 7$），那么从坐标 $3$ 到 $10$ 这一段的距离就一定是 $7$。”

1.  **前缀和（里程计）**：实时记录从起点到现在的总进度。
2.  **哈希表（记录仪）**：把走过的每一个“里程点”都记在账本上，并记录这个点出现了多少次（因为有负数和 $0$，同一个坐标可能会经过多次）。
3.  **瞬间匹配**：每走一步，就查账本：“以前有没有出现过 `current_sum - K`？”

---

### 二、 角色分配：三个关键变量

以 Canvas 示例中的 `nums = [3, 4, 7, -2, 2, 1], k = 7` 为例：

#### 1. `prefix` (当前里程)
* **任务**：记录现状。
* 它是从 `index 0` 到当前位置的实时累加和。

#### 2. `target = prefix - k` (时空坐标)
* **任务**：寻找“影子”。
* 我们不直接找 $K$，而是找“以前那个能让我们和现在相差 $K$ 的点”。

#### 3. `sum_dict` (历史账本)
* **任务**：存储过去。
* **初始值 `{0: 1}` 的意义**：这代表“在出发前，我们在坐标 $0$ 处待过 $1$ 次”。如果没有它，如果第一个数刚好是 $7$，你就找不到这个子数组了。

---

### 三、 算法精髓：为什么是 $O(N)$？

如果不使用哈希表，你每到一个新位置都要用 $O(N)$ 的时间往回看所有的区间（双重循环），总时间是 $O(N^2)$。
使用了哈希表后：
* **存入**：$O(1)$
* **查找**：$O(1)$
* **结果**：你只需要带着一个账本走一次数组，就能在行走的过程中顺便把所有符合条件的“历史区间”全部收割。

---

### 四、 过程模拟 (Canvas 示例核心片段)

1.  **初始**：`sum_dict = {0: 1}`。
2.  **遇到 3**：`prefix=3`。查 `3-7 = -4`。不在。存入 `3`：`{0:1, 3:1}`。
3.  **遇到 4**：`prefix=7`。查 `7-7 = 0`。**命中 0！** 计数加 1（子数组 `[3, 4]`）。存入 `7`：`{0:1, 3:1, 7:1}`。
4.  **遇到 7**：`prefix=14`。查 `14-7 = 7`。**命中 7！** 计数加 1（子数组 `[7]`）。存入 `14`：`{..., 7:1, 14:1}`。
5.  **遇到 -2**：`prefix=12`。查 `12-7 = 5`。不在。存入 `12`。
6.  **遇到 2**：`prefix=14`。查 `14-7 = 7`。**又命中 7！** 计数加 1。更新 `14`：`{..., 14:2}`。

---

### 五、 总结金句

> **“前缀和负责‘创造’坐标，哈希表负责‘管理’坐标。通过计算 `当前 - 过去 = K`，我们把复杂的区间搜索，变成了一个简单的填空题。”**
