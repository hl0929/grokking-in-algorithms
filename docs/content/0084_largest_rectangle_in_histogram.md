```python
def largest_rectangle_area(heights):
    heights = [0] + heights + [0]
    stack = []
    max_area = 0
    for i, h in enumerate(heights):
        while stack and h < heights[stack[-1]]:
            height = heights[stack.pop()]
            width = i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area


heights = [2,1,5,6,2,3]  # 10
result = largest_rectangle_area(heights)
print(result)
```

这道题目（LeetCode 84. 柱状图中最大的矩形）的核心思想是 **“单调递增栈（Monotonic Increasing Stack）”**。它是单调栈家族里最经典、最精妙的应用之一。

如果说“每日温度”是在找未来的救星，那么这道题就是在寻找每一根柱子的 **“左右边界”**。

---

### 一、 核心思想：寻找左右“护法”

想象你要以某一根柱子 `i` 为**高度**，向左右两边尽可能地延伸来构成长方形：
1.  **左边界**：左边第一个比它**矮**的柱子。
2.  **右边界**：右边第一个比它**矮**的柱子。
3.  **计算**：只要找到了这两根“矮柱子”，中间剩下的宽度就是这根柱子能撑起的最大地盘。

**哨兵机制（Sentinels）**：
我们在原数组前后各加一个 `0`。
* **左边的 0**：保证栈永远不会空，方便计算宽度。
* **右边的 0**：保证遍历结束时，栈里所有还在“怀才不遇”的柱子都能被强行弹出并计算。



---

### 二、 算法逻辑：维护上升趋势

#### 1. 变量准备
* `stack`：存放柱子的**下标**。我们强制让栈里的柱子高度保持 **单调递增**。
* `max_area`：全局最大面积记录。

#### 2. 触发计算（while 循环）
* 当我们遇到一根**比栈顶矮**的柱子 `h` 时，平衡被打破了！
* 此时，栈顶那根柱子（`stack.pop()`）终于找到了它的**右边界**（就是当前的 `i`）。
* 而它的**左边界**，就是它弹出来后，新的栈顶元素。
* **面积 = 弹出柱子的高度 × (右边界下标 - 左边界下标 - 1)**。

#### 3. 入栈
* 如果当前的柱子比栈顶高，说明它还有向右延伸的潜力，暂时入栈等待。

---

### 三、 过程模拟 (`heights = [2, 1, 5, 6, 2, 3]`)

加上哨兵变为 `[0, 2, 1, 5, 6, 2, 3, 0]`：
1.  **遇到 2**：入栈。
2.  **遇到 1**：1 < 2！触发计算。
    * 高度 = 2，左边界是 0（下标0），右边界是 1（下标2）。宽度 = 2-0-1 = 1。面积 = 2。
3.  **遇到 5, 6**：一路走高，全部入栈。`stack = [0, 2, 3, 4]` (对应值 0, 1, 5, 6)。
4.  **遇到 2**：2 < 6！触发计算。
    * 先弹 6：高度 6，宽度 5-3-1 = 1。面积 6。
    * 再弹 5：高度 5，宽度 5-2-1 = 2。面积 10。**（目前最大！）**
    * 遇到 1 < 2，停止弹出，2 入栈。
5.  **最后遇到末尾的 0**：把栈里剩下的 3、2 全部弹出结算。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 尽管有嵌套循环，但每根柱子只入栈一次、出栈一次。这是一个标准的线性时间算法。
* **空间复杂度：$O(N)$**
    * 最坏情况下（高度单调递增），栈的大小会达到 $N$。

---

### 五、 总结金句

> **“这叫‘急流勇退，借力打力’。单调栈在这里化身为一个动态的区间尺。每当我们遇到一根‘矮柱子’，它就像是一个哨信号，触发了对过去一段辉煌（递增区间）的结算。通过左右哨兵的守护，我们确保了每一根柱子都能在最合适的时机，找到属于自己的左右边界。这种利用高度差触发计算的思维，是解决几何区间问题的核心套路。”**
