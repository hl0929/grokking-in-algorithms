```python
def coin_change(coins, amount):
    if amount <= 0:
        return 0
    dp = [float("inf")] * (amount + 1)
    dp[0] = 0
    for i in range(1, amount + 1):
        for coin in coins:
            if i >= coin:
                dp[i] = min(dp[i], 1 + dp[i - coin])
    return -1 if dp[amount] == float("inf") else dp[amount]


coins = [1, 2, 5] 
amount = 11 # 3
result = coin_change(coins, amount)
print(result)
```


这道题目（LeetCode 322. 零钱兑换）的核心思想是 **“最优子结构（Optimal Substructure）”**。它是动态规划中经典的“完全背包问题”变种。

如果说“打家劫舍”是在做二选一，那么“零钱兑换”就是在 **“广撒网，选最强”**——在多种硬币组合中，挑出那个能让硬币数量最少的“最优解”。

---

### 一、 核心思想：层层递进的“拼图”

想象你要凑齐 `amount` 块钱。当你手里拿着一枚面值为 `coin` 的硬币时，你面临的问题就变成了：**“凑齐 `amount - coin` 块钱最少需要几枚硬币？”**

1.  **全局观**：要凑齐 11 块，我可以先放一个 5 块，然后看凑齐 6 块最少要多少枚。
2.  **不贪心**：有时候面值大的硬币未必是最佳选择。我们不能像贪心算法那样只选大的，而是要把所有可能的硬币都试一遍。
3.  **状态记录**：我们用一个 `dp` 数组记下从 1 块到 `amount` 块，每一级所需的最少硬币数。



---

### 二、 算法逻辑：从小目标到大梦想

#### 1. 初始化“无限大”
* `dp = [inf] * (amount + 1)`
* 为什么是无限大？因为我们要找的是 `min`（最小值）。还没凑出来之前，我们假设它是不可达到的。
* `dp[0] = 0`：凑齐 0 块钱，只需要 0 枚硬币。这是所有逻辑的起点。

#### 2. 双重循环的博弈
* **外层循环**：遍历每一个金额 `i`（从 1 块攒到 `amount` 块）。
* **内层循环**：尝试每一枚硬币 `coin`。
* **状态转移**：`dp[i] = min(dp[i], 1 + dp[i - coin])`
    * 这行代码的意思是：如果我用了这枚 `coin`，那总硬币数就是 `1`（当前这枚）加上 `凑齐剩下钱数所需的最少硬币数`。

#### 3. 最终判定
* 如果 `dp[amount]` 还是无限大，说明这道题无解（比如硬币只有 2 块，却要凑 3 块），返回 `-1`。

---

### 三、 过程模拟 (`coins = [1, 2, 5], amount = 11`)

1.  **凑 1 块**：只能选硬币 1。`dp[1] = 1 + dp[0] = 1`。
2.  **凑 2 块**：
    * 选硬币 1：`1 + dp[1] = 2`
    * 选硬币 2：`1 + dp[0] = 1`
    * `min(2, 1) = 1`。所以 `dp[2] = 1`。
3.  **依此类推...**
4.  **凑 11 块**：
    * 方案 A：选硬币 1，剩下凑 10 块（`1 + dp[10]`）
    * 方案 B：选硬币 2，剩下凑 9 块（`1 + dp[9]`）
    * 方案 C：选硬币 5，剩下凑 6 块（`1 + dp[6]`）
    * 比较发现，方案 C 或方案 B 的某些路径最终会导向 **3** 枚硬币（5+5+1）。

---

### 四、 复杂度分析

* **时间复杂度：$O(amount \times n)$**
    * 其中 $n$ 是硬币的种类数。虽然是双重循环，但逻辑非常清晰，是标准的线性时间增长。
* **空间复杂度：$O(amount)$**
    * 我们开辟了一个长度为 `amount + 1` 的数组来记录中间状态。

---

### 五、 总结金句

> **“这叫‘步步为营，博采众长’。零钱兑换不再是一次性的拍脑袋决策，而是把大额目标拆解成无数个小额目标。每一枚硬币的加入，都是在向前人的经验（已算出的 `dp` 值）寻求帮助。通过这种地毯式的搜索，我们终能从千万种组合中，精准锁定那条最短的捷径。”**
