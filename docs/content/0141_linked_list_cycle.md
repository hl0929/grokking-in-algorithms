class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
        
        
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False

这道题目（LeetCode 141. 环形链表）的核心思想依然是 **“弗洛伊德判圈算法（Floyd's Cycle-Finding Algorithm）”**。

如果说“删除倒数第 N 个”是利用身位差进行定位，那么这道题就是纯粹的 **“速度差追逐战”**。

---

### 一、 核心思想：操场上的圈套

想象一下，如果有两名运动员在跑道上跑步：
1.  **直线跑道**：如果跑道是直的，速度快的那个人（兔子）会永远领先，两人距离越来越远，直到兔子跑出终点。
2.  **环形操场**：如果跑道是一个圈，只要时间足够长，速度快的兔子一定会从后面 **“套圈”** 慢的那个人（乌龟）。

在链表中，如果 `fast` 指针最终遇到了 `slow` 指针，那就说明这个链表里藏着一个环。

---

### 二、 算法逻辑：快慢指针赛跑



#### 1. 裁判入场（初始化）
* 我们让 `slow` 和 `fast` 都从 `head` 出发。此时比赛正式开始。

#### 2. 步频设定
* **乌龟 (slow)**：每次走 **1** 步。
* **兔子 (fast)**：每次走 **2** 步。
* *逻辑点：兔子跑得快，它是去前方“侦查”有没有悬崖（链表终点 `None`）的。*

#### 3. 追逐过程
* 在每一个循环中，兔子先跳两下，乌龟慢悠悠挪一下。
* **终点判定**：如果兔子（或它的下一步）遇到了 `None`，说明前方是断头路，没有环。
* **相遇判定**：如果在某一时刻，`slow == fast`，说明兔子已经绕了一圈回来把乌龟给撞倒了。

---

### 三、 过程模拟 (`1 -> 2 -> (回到1)`)

1.  **初始**：`slow` 在 1，`fast` 在 1。
2.  **第一轮**：
    * `slow` 挪一步，到达 **2**。
    * `fast` 跳两步（1 → 2 → 1），到达 **1**。
    * 两人没撞上，继续。
3.  **第二轮**：
    * `slow` 挪一步（2 → 1），到达 **1**。
    * `fast` 跳两步（1 → 2 → 1），到达 **1**。
    * **命中！** `slow == fast`，裁判判定：有环！

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 如果没有环，兔子走到头就结束了。
    * 如果有环，兔子和乌龟的相对速度是 1 步/回合，兔子会在 $O(N)$ 的步数内追上乌龟。
* **空间复杂度：$O(1)$**
    * 我们只用了两个指针，没有使用哈希表（虽然哈希表也能解，但那需要 $O(N)$ 空间）。

---

### 五、 总结金句

> **“在无限的循环中，重逢是唯一的必然。我们派出一快一慢两名特种兵，慢的那个负责稳步推进，快的那个负责在前方开路。如果世界是平坦的直线，他们终将永别；如果世界是个圆圈，他们定会重逢。这就是快慢指针处理环路问题的浪漫逻辑。”**
