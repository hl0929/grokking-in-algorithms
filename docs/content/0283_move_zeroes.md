
[283. Move Zeroes](https://leetcode.cn/problems/move-zeroes)

[0283_move_zeroes](./html/0283_move_zeroes.html ":include :type=iframe")


```python
def move_zeroes(nums):
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow] = nums[fast]
            slow += 1
    for i in range(slow, len(nums)):
        nums[i] = 0
    
nums = [0,1,0,3,12] # [1,3,12,0,0]
move_zeroes(nums)
print(nums)
```

这道题目（LeetCode 283. 移动零）的核心思想是 **“双指针覆盖与填充”**，通过 **一次快慢指针扫描 + 一次尾部填充** 将 $O(N^2)$ 的数组平移优化为 $O(N)$ 的线性压实。


### 一、 核心思想：把“空格”挤到最后

想象你正在排队，队伍中间有几个人中途离开了（变成了 0），留下了空格。
这个算法的逻辑是：**“后面的人不需要原地等待，也不需要一个一个互相交换，而是直接看到前面有空位就坐过去。”**

1.  **快指针 `fast`（搜寻者）**：负责在前面跑，寻找所有非零的“活人”。
2.  **慢指针 `slow`（待填位）**：指向当前队伍最前面的那个“空位”。
3.  **覆盖而非交换**：发现非零就直接“覆盖”到 `slow` 的位置，而不是慢悠悠地做交换。

---

### 二、 角色分配：两个“搬运工”

以 `nums = [0, 1, 0, 3, 12]` 为例：

#### 1. `fast` 搜寻员（寻找非零）
* 它的任务是：**“滤掉所有的 0”**。
* 只要发现 `nums[fast] != 0`，就立刻喊话：“这里有一个非零数，赶快搬到前面的空位去！”

#### 2. `slow` 记录员（维护边界）
* 它的任务是：**“守住非零区域的末尾”**。
* 只有当一个非零数搬过来后，它才会向后移动一格，腾出下一个位置。

---

### 三、 算法精髓：两步走战略

这是该算法最稳健的地方：

* **第一步：压实 (Compression)**
    * 不管 0，只管把非零的数按顺序从 `0` 索引开始排。
    * 遍历结束时，数组前 `slow` 个元素全是按原顺序排列的非零数。
* **第二步：清零 (Zero-fill)**
    * 既然非零的都搬走了，剩下的坑位（从 `slow` 到数组末尾）肯定全是 `0`。
    * 直接一个循环，把后面全部刷成 `0`。

---

### 四、 过程模拟 (`nums = [0, 1, 0, 3, 12]`)

1.  **遇到 0**：`fast` 跳过。
2.  **遇到 1**：`nums[slow] = 1`（把 1 搬到索引 0），`slow` 变成 1。
3.  **遇到 0**：`fast` 跳过。
4.  **遇到 3**：`nums[slow] = 3`（把 3 搬到索引 1），`slow` 变成 2。
5.  **遇到 12**：`nums[slow] = 12`（把 12 搬到索引 2），`slow` 变成 3。
6.  **最后**：从 `slow` (索引 3) 到末尾全部填 `0`。数组变成 `[1, 3, 12, 0, 0]`。

---

### 五、 复杂度分析

* **时间复杂度：$O(N)$**
    * 虽然有两个循环，但每个位置最多被访问两次，依然是线性时间。
* **空间复杂度：$O(1)$**
    * 原地修改，没有使用额外数组。

---

### 六、 总结金句

> **“快指针搜寻非零值，慢指针按序排坑位。先把好数往前凑，再把余位全填零。”**
