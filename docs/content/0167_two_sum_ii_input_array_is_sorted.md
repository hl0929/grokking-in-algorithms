```python
def two_sum(numbers, target):
    left, right = 0, len(numbers) - 1
    while left < right:
        total = numbers[left] + numbers[right]
        if total == target:
            return [left + 1, right + 1]
        elif total > target:
            right -= 1
        else:
            left += 1


numbers = [2,3,5]  
target = 6 # [1,3]
result = two_sum(numbers, target)
print(result)
```
这道题目（LeetCode 167. 两数之和 II - 输入有序数组）的核心思想是 **“有序约束下的对撞双指针（Two Pointers Collision）”**。

如果说普通的“两数之和”是在哈希表里玩“捉迷藏”，那么这道题就是在利用数组的**单调性**进行一场精准的“包围战”。

---

### 一、 核心思想：有序带来的“上帝视角”

这道题最关键的预设是：**数组已经排好序了**。
在一个有序数组中，当你把两端的数字相加时，你其实掌握了某种“方向盘”：
1.  **和太大了**：如果你想让和变小，唯一的方法就是把右边的指针向左移（找个更小的数）。
2.  **和太小了**：如果你想让和变大，唯一的方法就是把左边的指针向右移（找个更大的数）。
3.  **精准命中**：这种单调性让你不需要暴力枚举所有组合，每一步都在缩小搜索范围。

---

### 二、 算法逻辑：双指针的对撞

#### 1. 阵地排布
* `left` 指向数组最左端（最小值）。
* `right` 指向数组最右端（最大值）。

#### 2. 动态调控（While 循环）
* 算出当前两个“极值”的和 `total`：
    * **相等**：皆大欢喜，直接返回下标。注意题目要求返回的是从 `1` 开始计数的索引，所以要 `+1`。
    * **大于 `target`**：说明目前的“火力”太猛了，最大值 `numbers[right]` 太大了，必须弃车保帅，`right -= 1`。
    * **小于 `target`**：说明目前的“火力”不足，最小值 `numbers[left]` 拖了后腿，必须补强，`left += 1`。

#### 3. 结果保证
* 题目保证了有且仅有一个答案，所以当 `left` 和 `right` 相遇前，真相一定会浮出水面。

---

### 三、 过程模拟 (`numbers = [2, 3, 5], target = 6`)

1.  **初始状态**：`left = 0 (2)`, `right = 2 (5)`。
    * `total = 2 + 5 = 7`。
    * `7 > 6`：太大了！右指针左移。
2.  **第二步**：`left = 0 (2)`, `right = 1 (3)`。
    * `total = 2 + 3 = 5`。
    * `5 < 6`：太小了！左指针右移。
3.  **第三步**（本示例针对原题逻辑稍作调整）：
    * 实际上，对于 `[2, 3, 5]` 找 `6`，代码会继续移动直到找到匹配。
    * 如果 `target = 8`，第一步 `2+5=7 < 8`，左移变为 `3+5=8`，命中！返回 `[2, 3]`。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 最坏情况下，两个指针共同走完整个数组。比起暴力枚举的 $O(N^2)$，效率有了质的飞跃。
* **空间复杂度：$O(1)$**
    * 只用了两个指针变量，完全不依赖额外空间。

---

### 五、 总结金句

> **“这叫‘顺势而为，两端包抄’。有序数组本身就自带了逻辑的阶梯。通过两端对撞的指针，我们避开了无意义的盲目搜索。每一次指针的位移，都是对单调性规律的一次完美服从。这种利用有序性进行范围缩减的思路，是二分查找之外，处理有序序列问题最灵动的方式。”**
