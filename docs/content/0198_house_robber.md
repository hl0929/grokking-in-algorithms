
[198. House Robber](https://leetcode.cn/problems/house-robber)

[0198_house_robber](./html/0198_house_robber.html ":include :type=iframe")

```python
def rob(nums):
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]
    prev2 = nums[0]
    prev1 = max(nums[0], nums[1])
    for i in range(2, len(nums)):
        curr = max(prev1, prev2 + nums[i])
        prev2 = prev1
        prev1 = curr
    return prev1


nums = [1,2,3,1] # 4
result = rob(nums)
print(result)
```

这道题目（LeetCode 198. 打家劫舍）的核心思想是 **“收益博弈（Profit Trade-off）”**。它是动态规划（DP）中非常经典的一类问题：如何在满足特定约束条件下，榨取最大的价值。

如果说“爬楼梯”是简单的累加，那么“打家劫舍”就是一场 **“二选一的权衡游戏”**。

---

### 一、 核心思想：贪婪中的克制



在这个街道上，你面临一个硬性约束：**不能抢劫相邻的两家**（否则报警系统会响）。当你走到第 $i$ 家店时，你只有两个选择：

1.  **抢这一家**：那么你一定不能抢第 $i-1$ 家。你的总收益 = **前前一家的最大收益 + 这一家的钱**。
2.  **不抢这一家**：那你的收益就直接等于 **前一家的最大收益**（哪怕前一家其实也没抢）。

**结论：** 每一家的最大收益，取决于“抢它”和“不抢它”哪个钱更多。

---

### 二、 算法逻辑：状态的滚动与更替

#### 1. 初始决策
* `prev2`：代表抢到“前前一家”时的最大金额。
* `prev1`：代表抢到“前一家”时的最大金额（或者是前两家中的较大值）。

#### 2. 状态转移方程
* `curr = max(prev1, prev2 + nums[i])`
* 这个公式就是灵魂：它是 **“维持现状”** 与 **“另辟蹊径（跳过一家抢大的）”** 之间的对决。

#### 3. 滚动优化
* 我们同样不需要 $O(N)$ 的数组。通过 `prev2 = prev1` 和 `prev1 = curr`，我们像滚动轮子一样，只保留最有用的两个信息，向街道深处推进。

---

### 三、 过程模拟 (`nums = [1, 2, 3, 1]`)

1.  **初始化**：
    * `prev2 = 1` (第 0 家)
    * `prev1 = max(1, 2) = 2` (前两家最大能拿 2)
2.  **计算第 2 家 (金额 3)**：
    * 抢：`prev2 + 3 = 1 + 3 = 4`
    * 不抢：`prev1 = 2`
    * `max(4, 2) = 4`。此时 `prev2` 变为 2, `prev1` 变为 4。
3.  **计算第 3 家 (金额 1)**：
    * 抢：`prev2 + 1 = 2 + 1 = 3`
    * 不抢：`prev1 = 4`
    * `max(3, 4) = 4`。最终 `prev1` 还是 4。
4.  **返回结果**：4。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 无论街道多长，我们只需单向走一遍，每家只做一次简单的数学比大小。
* **空间复杂度：$O(1)$**
    * 我们没有使用额外的记录表，仅仅用了几个变量来记录状态的更迭，内存效率极高。

---

### 五、 总结金句

> **“这叫‘步步博弈，全局最优’。算法在这里不再是死板的数学公式，而是一个精明的决策者。它在每一家门口都会停下来算一笔账：是守着现在的钱不动，还是冒着放弃上一家的风险去抢下这一家？这种局部最优的不断演进，最终构建出了整条街道的最优解。”**
