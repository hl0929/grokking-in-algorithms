
[128. Longest Consecutive Sequence](https://leetcode.cn/problems/longest-consecutive-sequence)

[0128_longest_consecutive_sequence](./html/0128_longest_consecutive_sequence.html ":include :type=iframe")

```python
def longest_consecutive_sequence(nums):
    num_set = set(nums)
    max_len = 0
    for num in num_set:
        if num - 1 not in num_set:
            length = 1
            cur = num
            while cur + 1 in num_set:
                cur += 1
                length += 1
            max_len = max(max_len, length)
    return max_len


nums = [100,4,200,1,3,2] # 4
result = longest_consecutive_sequence(nums)
print(result)    
```

这道题目（LeetCode 128. 最长连续序列）的核心思想是 **“去重哈希 + 寻找起点”**，它通过 **哈希表（Hash Set）** 将原本需要排序才能解决的 $O(N \log N)$ 问题，优化为了 **$O(N)$ 的线性扫描**。


### 一、 核心思想：只找“领头羊”

想象一下，有一群乱序的数字，里面藏着几条连续的“龙”（比如 `1-2-3-4`）。如果你从每个数字都开始往后数，会做大量重复劳动。

算法的逻辑是：**“我只从每一条龙的头开始数。如果一个数有左邻居，说明它只是个‘小兵’，直接跳过；如果没有左邻居，它就是‘队长’，从它开始接龙。”**

1.  **空间换时间**：先把数组存入 `Set`，利用其 $O(1)$ 的查找特性，快速判断“邻居”在不在。
2.  **过滤冗余**：通过判断 `num - 1` 是否存在，瞬间排除掉所有非起点的数字。
3.  **线性收割**：虽然有嵌套的 `while` 循环，但每个数字在整个过程中只会被“数”到常数次。

---

### 二、 角色分配：两个“侦察兵”

以 `nums = [100, 4, 200, 1, 3, 2]` 为例：

#### 1. `num - 1` 探测员（寻找起点）
* 它的任务是：**“向上溯源”**。
* 当看到 `4` 时，它问 `Set`：“`3` 在吗？”
* `Set` 回答：“在。”
* 探测员：“好，那 `4` 不是起点，咱们别在它身上浪费时间。”

#### 2. `num + 1` 接龙员（测量长度）
* 它的任务是：**“向下延伸”**。
* 当看到 `1` 时，它问 `Set`：“`0` 在吗？”
* `Set` 回答：“不在。”
* 接龙员：“太棒了！`1` 是起点。快看看 `2, 3, 4...` 在不在，数数这条龙有多长。”

---

### 三、 算法精髓：为什么是 $O(N)$？

这是最违反直觉的地方。虽然代码看起来是“循环套循环”，但：
* **外层 `for` 循环**：遍历每个数字。
* **内层 `while` 循环**：只有当 `num` 是起点时才执行。
* **结果**：数组里的每个数字，要么在 `if` 阶段被跳过，要么在 `while` 阶段被数一次。**没有一个数字会被数两次。**

---

### 四、 过程模拟 (`nums = [100, 4, 200, 1, 3, 2]`)

1.  **遇到 100**：`99` 不在，它是起点。接龙查找 `101`，不在。长度为 1。
2.  **遇到 4**：`3` 在 `Set` 里，**跳过**（因为它不是起点）。
3.  **遇到 200**：`199` 不在，它是起点。接龙查找 `201`，不在。长度为 1。
4.  **遇到 1**：`0` 不在，**它是起点！**
    * 查 `2`，在。
    * 查 `3`，在。
    * 查 `4`，在。
    * 长度记为 4。
5.  **遇到 3**：`2` 在 `Set` 里，**跳过**。
6.  **遇到 2**：`1` 在 `Set` 里，**跳过**。

---

### 五、 复杂度分析

* **时间复杂度：$O(N)$**
    * 构建 `Set` 需要 $O(N)$。
    * 遍历数组寻找起点并接龙，每个数字最多被访问 2 次。
* **空间复杂度：$O(N)$**
    * 需要一个哈希表来存储所有数字。
