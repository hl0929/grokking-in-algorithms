```python
def longest_common_subsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]


text1 = "abcde" 
text2 = "ace" # 3
result = longest_common_subsequence(text1, text2)
print(result)
```

这道题目（LeetCode 1143. 最长公共子序列，简称 **LCS**）的核心思想是 **“基因比对（Sequence Alignment）”**。它是动态规划（DP）中处理“双序列”问题的殿堂级模板。

如果说“零钱兑换”是在一条线上跳跃，那么 LCS 就是在 **“二维棋盘”** 上寻找一条连接两个灵魂的最长路径。

---

### 一、 核心思想：两个世界的碰撞

想象你有两条长度不一的项链（字符串），你想找出它们**共有且顺序一致**的最长部分：

1.  **字符匹配**：如果两个字符串末尾的字符刚好一样，那么“太棒了！最长长度直接加 1”，剩下的问题就变成了去掉这两个字符后的子问题。
2.  **字符不匹配**：如果末尾字符不一样，那就尴尬了。我们要么试试去掉 `text1` 的末尾，要么试试去掉 `text2` 的末尾，看看哪边的“后续潜力”更大（取 `max`）。



---

### 二、 算法逻辑：构建二维地图

#### 1. 建立空地（初始化）
* `dp[m+1][n+1]`：我们创建了一个二维网格。`dp[i][j]` 代表 `text1` 的前 `i` 个字符和 `text2` 的前 `j` 个字符能拼出的最长长度。
* **第一行和第一列**：全是 `0`。意思是如果其中一个字符串是空的，那公共部分长度自然是 `0`。

#### 2. 状态转移：棋盘上的抉择
* **命中（Matched）**：`text1[i-1] == text2[j-1]`。
    * 逻辑：`dp[i][j] = dp[i-1][j-1] + 1`。
    * 就像是在棋盘上**走对角线**：当前的成功是建立在两人都缩短一个字符的基础上的。
* **错过（Mismatched）**：`text1[i-1] != text2[j-1]`。
    * 逻辑：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。
    * 就像是在棋盘上**向左或向上**看：继承之前最强的那个状态。

#### 3. 结果产出
* 最终，网格右下角的那个数字 `dp[m][n]` 就是我们辛辛苦苦积累出来的“最大重合度”。

---

### 三、 过程模拟 (`text1="abcde", text2="ace"`)

1.  **比较 'a' 与 'a'**：匹配！`dp[1][1] = 0 + 1 = 1`。
2.  **比较 'b' 与 'c'**：不匹配。看左边或上边，继承之前的 `1`。
3.  **比较 'c' 与 'c'**：匹配！此时找到的是 `ac`，它是建立在 `a` 的基础上加 1。
4.  **最后...**：随着循环推进，`e` 与 `e` 再次碰撞，最长长度最终定格在 **3**。

---

### 四、 复杂度分析

* **时间复杂度：$O(M \times N)$**
    * $M$ 和 $N$ 是两个字符串的长度。你需要填满整个二维网格，每一个格子只需做一次简单的判断。
* **空间复杂度：$O(M \times N)$**
    * 我们开辟了一个二维数组。
    * *小贴士：其实可以利用“滚动数组”优化到 $O(\min(M, N))$，但面试时先把二维逻辑讲清楚是加分项。*

---

### 五、 总结金句

> **“这叫‘步步为营，跨时空寻亲’。LCS 不再关注局部，而是通过一张二维网格，记录了两个序列在每一个演化阶段的‘共同语言’。每一次字符的碰撞，都是对之前累积成果的肯定；每一次不匹配的让步，都是为了在未来的某个角落，遇见更长的重逢。”**
