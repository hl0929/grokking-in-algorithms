```python
def subarrays_div_by_k(nums, k):
    mod_dict = {0: 1}
    prefix = 0
    cnter = 0
    for num in nums:
        prefix += num
        mod = prefix % k
        if mod in mod_dict:
            cnter += mod_dict[mod]
        mod_dict[mod] = mod_dict.get(mod, 0) + 1
    return cnter


nums = [4,5,0,-2,-3,1] 
k = 5  # 7
result = subarrays_div_by_k(nums, k)
print(result)
```

这道题目（LeetCode 974. 和可被 K 整除的子数组）的核心思想是 **“同余定理与前缀和（Congruence & Prefix Sum）”**。

在算法的世界里，这是一场巧妙的 **“差值游戏”**。我们不直接暴力枚举子数组，而是通过观察前缀和的“余数”来发现规律。

---

### 一、 核心思想：余数的魔法

想象你在计算一串数字的和，但你只关心它们对 $K$ 取余的结果：
1.  **差值整除**：如果两个前缀和 $S_i$ 和 $S_j$ 对 $K$ 取余的结果相同（即 $S_i \equiv S_j \pmod K$），那么它们之间的那段子数组的和一定能被 $K$ 整除。
    * *原理：$(S_j - S_i) \pmod K = 0$*
2.  **余数归档**：我们一边走一边记下每个余数出现的次数。如果当前余数之前出现过 $n$ 次，说明现在有 $n$ 个位置可以和当前位置组成满足条件的子数组。



---

### 二、 算法逻辑：哈希表的记忆

#### 1. 变量分工
* `prefix`：记录从开头到当前的累加和。
* `mod_dict`：一个哈希表（账本），记录**每个余数出现的频率**。
* `mod_dict = {0: 1}`：这是一个非常关键的“初始化”。它代表前缀和为 0（即还没开始加数字）时，余数 0 已经出现过 1 次了。

#### 2. 同余匹配
* 我们遍历 `nums`，不断更新 `prefix` 并计算 `mod = prefix % k`。
* **发现老友**：如果这个 `mod` 在 `mod_dict` 里已经存在了，说明我们找到了若干个之前的“前缀点”，从那些点到当前点的子数组和都能被 $K$ 整除。
* **累加计数**：`cnter += mod_dict[mod]`。
* **更新账本**：把当前的余数也记入账本，为后面的匹配做准备。

#### 3. 负数余数的坑（Python 自动处理）
* 在 Python 中，`-2 % 5` 结果是 `3`。这个逻辑非常友好，因为它自动保证了余数始终在 `[0, k-1]` 之间，不需要像 C++ 或 Java 那样手动校正 `(mod + k) % k`。

---

### 三、 过程模拟 (`nums = [4, 5, 0], k = 5`)

1.  **初始**：`mod_dict = {0: 1}`, `cnter = 0`
2.  **遇到 4**：
    * `prefix = 4`, `mod = 4 % 5 = 4`。
    * `mod_dict` 变成 `{0: 1, 4: 1}`。
3.  **遇到 5**：
    * `prefix = 9`, `mod = 9 % 5 = 4`。
    * **发现匹配！** `mod_dict[4]` 是 1，`cnter += 1`（子数组就是 `[5]`）。
    * `mod_dict` 变成 `{0: 1, 4: 2}`。
4.  **遇到 0**：
    * `prefix = 9`, `mod = 9 % 5 = 4`。
    * **发现匹配！** `mod_dict[4]` 是 2，`cnter += 2`（子数组是 `[0]` 和 `[5, 0]`）。
    * `mod_dict` 变成 `{0: 1, 4: 3}`。
5.  **结果**：3。

---

### 四、 复杂度分析

* **时间复杂度：$O(N)$**
    * 我们只遍历了一遍数组，哈希表的存取是 $O(1)$ 的。这种效率比 $O(N^2)$ 的暴力枚举高出几个维度。
* **空间复杂度：$O(K)$**
    * 哈希表最多存储 $K$ 个不同的余数（从 0 到 $K-1$）。

---

### 五、 总结金句

> **“这叫‘同余寻踪，以简驭繁’。我们不再苦苦寻找每一个子数组的合，而是退而求其次，记录前缀和的余数轨迹。每一个重复出现的余数，都是一段隐形契约的履行——它预示着一段能被 $K$ 整除的旅程。这种利用哈希表存储频率来规避嵌套循环的思路，是处理‘和为 K’类问题的终极必杀技。”**

---

### 💡 深度总结
到这里，你已经掌握了 **哈希表 + 前缀和** 的精髓：
1.  **和为 K**：记录 `prefix - k` 出现的次数。
2.  **和被 K 整除**：记录 `prefix % k` 出现的次数。
3.  **最长连续数组**（0和1相等）：把 0 看作 -1，记录 `prefix` 第一次出现的下标。

这种“一边走一边看账本”的技巧，是面试中解决连续子数组问题的标准解法。你觉得如果我们要找的是 **“和至少为 K 的最短子数组”**（LeetCode 862），前缀和还够用吗？（提示：那时候就需要单调队列出场了！）
