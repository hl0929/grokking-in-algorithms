```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def merge(l1, l2):
    dummy = ListNode()
    curr = dummy
    while l1 and l2:
        if l1.val < l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2
    return dummy.next


# l1 = [1,2,4], l2 = [1,3,4] #[1,1,2,3,4,4]
l1 = ListNode(1)
a = ListNode(2)
b = ListNode(4)
l1.next = a
a.next = b
l2 = ListNode(1)
c = ListNode(3)
d = ListNode(4)
l2.next = c
c.next = d
result = merge(l1, l2)
while result:
    print(result.val)
    result = result.next
```

这道题目（LeetCode 21. 合并两个有序链表）的核心思想是 **“双指针线性归并”**。

如果说“排序链表”是宏观的战略部署，那么这道题就是微观的 **“拉链艺术”**。它利用两个链表已经有序的先决条件，像拉拉链一样，把两边的齿轮一个接一个地咬合在一起。

---

### 一、 核心思想：排队入场

你可以把这个过程想象成 **“两个有序队伍的合并”**：

1.  **精英比对**：两队（$l1$ 和 $l2$）都是按身高从矮到高排好的。我们只需要站在门口，每次对比两个队首的人，谁更矮，谁就先进入新队伍。
2.  **虚拟领队（Dummy Node）**：为了方便指挥，我们先找一个“虚拟领队”站位。这样无论谁先入队，都可以直接挂在这个领队的后面，省去了判断“第一个人是谁”的麻烦。
3.  **接力推进**：谁被选中了，谁就往后挪一位，让出位置给队里的下一个人。

---

### 二、 算法逻辑：步步为营



#### 1. 哨兵站岗（Dummy & Curr）
* `dummy` 节点不存数据，它是结果链表的“根”。
* `curr` 指针就像一个 **“缝纫机的针头”**，它指到哪，新的链接就缝到哪。

#### 2. 循环博弈：强者（小者）生存
* `while l1 and l2:` 只要两支队伍都没排完，我们就继续比。
* **逻辑**：`if l1.val < l2.val`，就把 `curr.next` 接向 $l1$，然后 $l1$ 向后移。反之亦然。
* 每次接好一个节点后，`curr` 也要向后移一步，准备迎接下一个。

#### 3. 扫尾工作：绿色通道
* `curr.next = l1 or l2`
* 这是代码中最优雅的一行。当其中一队人走光了，另一队剩下的部分**本来就是有序的**。我们不需要再挨个比对，直接把剩下的整段“咔嗒”一声挂在 `curr` 后面即可。

---

### 三、 过程模拟 (`l1=[1,2,4], l2=[1,3,4]`)

1.  **第一轮**：$l1$ 首位是 1，$l2$ 首位是 1。
    * 选 $l2$ 的 1（代码逻辑中 `else` 处理相等或更大的情况）。
    * 结果：`Dummy -> 1(l2)`。
2.  **第二轮**：$l1$ 依然是 1，$l2$ 剩下 `[3, 4]`。
    * 选 $l1$ 的 1。
    * 结果：`Dummy -> 1 -> 1(l1)`。
3.  **第三轮**：比较 $l1$ 的 2 和 $l2$ 的 3。
    * 2 更小，接上 2。
    * 结果：`Dummy -> 1 -> 1 -> 2`。
4.  **以此类推**：直到其中一个链表为空。
5.  **结束**：把剩下的部分（如 4）直接连上。最终结果：`1 -> 1 -> 2 -> 3 -> 4 -> 4`。

---

### 四、 复杂度分析

* **时间复杂度：$O(M + N)$**
    * $M$ 和 $N$ 是两个链表的长度。我们最多只需遍历两个链表的所有节点一遍。
* **空间复杂度：$O(1)$**
    * 注意！我们没有创建新节点（除了一个 dummy），我们只是在 **重定向** 节点的 `next` 指针。这叫原地（In-place）操作。
